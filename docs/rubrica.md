# Rubrica de evaluacion

| Criterio | Descripcion | Puntaje |
|---|---|---:|
| Producto ejecutable | El proyecto corregido se puede ejecutar siguiendo el README y genera el reporte esperado. | 25 |
| Correccion de errores | Identifica, corrige y documenta varios errores del script principal. | 20 |
| Evidencia de pruebas | Incluye resultados de ejecucion, pruebas finales y archivo de reporte generado. | 15 |
| Organizacion del proyecto | Mantiene una estructura clara con carpetas `data`, `src`, `output`, `docs` y `evidencias`. | 10 |
| Bitacora tecnica | Registra error, mensaje, causa, solucion y verificacion de cada correccion. | 15 |
| Comparacion tecnica del entorno | Analiza funcionalidad, compatibilidad y costo del entorno usado. | 10 |
| Coherencia contextual | La solucion se relaciona con el problema rural planteado. | 5 |
| Total |  | 100 |
